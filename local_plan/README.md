# local_plan ディレクトリ

このディレクトリには、test-kanteen の開発計画や設計ドキュメントを格納します。

## ファイル命名規則

```
YYYYMMDD_NN_title.md
```

- `YYYYMMDD`: 日付（例: 20251115）
- `NN`: 通し番号（01, 02, 03...）
- `title`: プランのタイトル（英数字とハイフン、小文字）

### 例
```
20251115_01_remove-coverage-gap-command.md
20251115_02_add-llm-guide.md
20251116_01_improve-extract-output.md
```

## プランニングファイルの構成

各プランニングファイルには以下のセクションを含めます：

```markdown
# [タイトル]

## 背景・課題

なぜこの変更が必要か

## 現状分析

現在の実装の問題点

## 提案

何をどう変更するか

## 実装計画

1. ステップ1
2. ステップ2
3. ...

## 影響範囲

- 変更が必要なファイル
- 削除するファイル
- テストへの影響

## 判断基準

- この変更を採用する理由
- 代替案との比較

## 次のステップ

実装後に必要な作業
```
